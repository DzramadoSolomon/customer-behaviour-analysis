# **📊 Customer Behaviour Analysis Dashboard**

## **🔍 Project Overview**

This project presents an end-to-end data analysis pipeline designed to uncover insights into customer shopping behavior. It integrates **Python for data preprocessing and exploratory analysis**, **SQL Server for business intelligence querying**, and **Power BI for interactive visualization**.

The goal is to transform raw transactional data into actionable insights that support data-driven decision-making.

---

## **🚀 Key Features**

* Data cleaning and preprocessing using Python
* Exploratory Data Analysis (EDA)
* SQL-based business problem solving (10 analytical questions)
* Interactive Power BI dashboard
* Customer segmentation and behavioral insights

## **📌 Business Questions Answered**

This project answers key analytical questions such as:

1. Revenue comparison between male and female customers
2. Customers who used discounts but spent above average
3. Top-rated products
4. Shipping method impact on spending
5. Subscription vs non-subscription spending behavior
6. Discount usage trends
7. Customer segmentation (New, Returning, Loyal)
8. Top products by category
9. Repeat customer subscription patterns
10. Revenue contribution by age group

---

## **🧹 Data Processing (Python)**

* Handled missing values and inconsistencies
* Standardized categorical variables
* Performed feature engineering
* Conducted exploratory data analysis (EDA)

---

## **🗄️ SQL Analysis**

* Aggregations (`SUM`, `AVG`, `COUNT`)
* Window functions and ranking
* Subqueries and conditional logic
* Data segmentation using `CASE WHEN`

---

## **📊 Dashboard Insights**

* Clothing is the top-performing category in both revenue and sales
* Majority of customers are non-subscribers (~73%)
* Young adults and middle-aged groups drive the most revenue
* Discount usage does not significantly reduce overall spending
* Repeat customers show higher engagement

## **⚡ How to Run the Project**

### **1. Python (Data Cleaning & EDA)**

```bash
cd notebooks
jupyter notebook data_cleaning_eda.ipynb
```

### **2. SQL Server**

* Import cleaned dataset into SQL Server
* Run queries from:

```
sql/business_queries.sql
```

### **3. Power BI**

* Open:

```
dashboard/customer_dashboard.pbix
```

* Refresh data if needed
